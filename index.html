A branch marks a point where the files in the repository diverged. Branches are a way to maintain alternate histories of a project you're working on. "git init" initializes a default branch called "master".

Create branches when:

  • You're preparing to make major changes to your project
  • Your're about to reset changes that you don't want to lose
  • You're going to work on a feature that may or may not end up being included in the main project.

Reasons to make new branches:

  • Experimental changes
  • New features
  • Bug fixes

Merging takes two or more branches and combines their history into one branch.
